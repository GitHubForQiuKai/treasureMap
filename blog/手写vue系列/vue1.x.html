# 前言

    • 支持v-text v-html

# 实现分析

主要实现包含以下几部分：

  • Quick：框架主体
  • Obsever：数据响应式（对象和数组）
  • Watcher：执行页面更新函数（dom操作）
  • Dep：收集管理多个Watcher、通知更新
  • Compiler：编译模板、生成更新函数、创建Watcher

# 数据响应式

数据响应式包括Object和Array：

  • 对于Object类型，会在defineReactive中采用Object.defineProperty将其转化为getter/setter。
  • 对于Array类型，会重写数组原型上的push, pop, shift ,unshift, splice, sort, reverse方法。

class Observer {

        if (Array.isArray(value)) {
            this.observeArray()
        } else {
            this.walk()
        }
    }

    walk() {}

    observeArray() {}
}

收集依赖和通知更新在后面complie和Dep的阶段完善

虽然Object.definePoproty完成了简单对象的响应式，但是还存在一些问题：

  1. 不支持复杂对象（嵌套对象）
  2. 不支持对象覆盖，比如obj[key] = {}

对于问题1，我们对其做递归操作；对于问题2，我们对新设置的值做响应式处理。






































 











 






class Observer {

    constructor(value) {
        this.value = value

        // 这里区分对象和数组
        if (Array.isArray(value)) {
            this.observeArray()
        } else {
            this.walk()
        }
    }

    walk() {
        for (const key in obj) {
            if (object.hasOwnProperty(key)) {
                Observer.deineReactive(obj, key, obj[key])
            }
        }
    }

    static observe(obj) {
        if(!obj || typeof obj !== 'object') return

        new Observer(obj)
    }

    /**
     * 响应式对象
     * @param {*} obj 目标对象
     * @param {*} key 目标键
     * @param {*} value 目标值
     */
    static deineReactive(obj, key, value) {
        if (Object.prototype.toString.call(obj) !== '[object Object]') return

        // 递归操作
        Observer.observe(value)

        Object.defineProperty(obj, key, {
            get() {
                // TODO 收集依赖

                return value
            },

            set(newValue) {
                // TODO 通知更新

                Observer.observe(newValue)// newValue可能为object类型，需进行响应式处理
                value = newValue
            }
        })
    }
}

# 数组响应式

对于数组的响应式，覆盖数组原型上的方法

    observeArray(value) {
        // 获取数组原型
        const originProto = Array.prototype;
        const arrayMethods = Object.create(originProto);
        
        ['push', 'pop', 'shift', 'unshift', 'splice', 'sort', 'reverse'].forEach(function (method) {
            // 修改原型方法
            arrayMethods[method] = function () {
                // 执行原始操作
                originProto[method].apply(this, arguments)

                // TODO 通知更新
                console.log('执行了方法' + method)
            }
        })

        // 覆盖实例上的原型为修改后的原型
        value.__proto__ = arrayMethods
    }

# 编译Compiler

Compiler编译是Vue框架中发动机，它是链接Observer、Wacther、Dep的发动者，从而促进形成整个mvvm框架的闭环。

主要包含以下功能：

  • 编译模板
  • 生成更新函数（初始化界面）
  • 创建Watcher

# 编译模板

  • 编译模板/html
    • 递归遍历子元素

      • # 元素节点

        • component组件
      • 文本节点

# vue1

  • data中的每一个属性key对应一个Dep
  • 界面中的每一个动态依赖对应一个Watcher

# vue2

  • data中的每一个属性key对应一个Dep
  • 每一个组件对应一个watcher
